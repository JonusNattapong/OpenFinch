# OpenFinch v0.1.0 Release Notes

**Release date:** 2025-05-08

## Summary

OpenFinch v0.1.0 is the initial public release of an open-source, self-hosted AI web agent infrastructure. It provides search, fetch, extract, browser automation, and autonomous agent capabilities through a unified REST API, CLI, SDK, MCP server, and web dashboard — all running on your own machine via Docker Compose.

## Installation

### Prerequisites

- Docker & Docker Compose
- Node.js 22+ (for CLI/SDK)
- 4GB+ RAM (8GB+ recommended for browser workloads)

### Quick Start

```bash
# Clone
git clone https://github.com/JonusNattapong/openfinch.git
cd openfinch

# Configure
cp .env.example .env
# Set at least one LLM API key in .env

# Start everything
docker compose up -d

# Verify
curl http://localhost:8787/health

# Run diagnostics
npx @openfinch/cli doctor
```

## Known Limitations

1. **Browser sessions are in-memory** — Sessions are stored in-memory on the API server. Restarting the API loses active sessions. Postgres persistence is planned.
2. **Agent runs are in-memory** — Agent run state and events are stored in-memory. Restarting the API loses active run data.
3. **No authentication** — The API has no built-in authentication. It's designed for local network use only. Do not expose to the public internet without a reverse proxy.
4. **Single-user** — The dashboard and API are single-user. Multi-user support is not implemented.
5. **No response caching via Postgres** — Caching uses Redis (primary) with in-memory fallback. Postgres-based caching is planned.
6. **No stealth/bot evasion** — OpenFinch identifies itself as a bot and respects robots.txt. It is not designed for adversarial scraping.
7. **Ollama requires host access** — In Docker Compose, Ollama access uses `host.docker.internal`. This may not work on all platforms.
8. **Windows PowerShell curl** — PowerShell's `curl` is an alias for `Invoke-WebRequest`. Use `curl.exe` or the CLI instead.

## Upgrade Notes

This is the first release. No upgrade path from previous versions exists.

## Verification Checklist

After deployment, verify:

```bash
# Check API
curl http://localhost:8787/health

# Run diagnostics
npx @openfinch/cli doctor

# Test search
curl -X POST http://localhost:8787/v1/search \
  -H 'Content-Type: application/json' \
  -d '{"query":"test","limit":1}'

# Test fetch
curl -X POST http://localhost:8787/v1/fetch \
  -H 'Content-Type: application/json' \
  -d '{"url":"https://example.com"}'

# Check providers
curl http://localhost:8787/v1/agent/providers
```
